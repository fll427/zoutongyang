## 🛠 工具篇：从环境搭建到大模型调用

### 1. Python 环境配置：别折腾，直接上 Anaconda
对于初学者，Python 环境配置就像是给手术室准备器械，乱放会导致手术失败。

* **快速上手**：直接在 B 站搜索“Anaconda 安装教程”，它是目前最省心的科学计算集成环境。
* **核心概念**：重点理解**“虚拟环境”**。简单来说，它能让你为不同项目创建独立的“房间”，避免不同版本的库产生冲突。
* **实操建议**：一定要亲自打开 `Navigator` 或 `Terminal`（终端），熟悉界面，尝试运行你的第一行代码：

```python
print("Hello World! 医学生跨界成功！")
```

想要调用大模型（LLM）实现自动化处理，**API Key** 就是你的通行证。

* **首推方案**：访问 **DeepSeek** 官网。其模型逻辑强，价格极低。
* **操作建议**：建议先充值 **1 元钱**。DeepSeek 兼容 OpenAI 格式，非常适合新手练手。
* **代码获取**：API 调用代码直接让 AI 帮你写，你只需负责运行。

> 绝对不要将你的 API Key 上传到 GitHub 或发在朋友圈！一旦泄露，余额可能瞬间被盗刷。

## 💻 实战篇：代码跑不通？AI 就是你的 24 小时助教

遇到报错（Traceback）不要慌，这在编程中是常态。学会用正确的 **Prompt（提示词）** 提问是关键。

### 1. 高效 Debug 的“三步提问法”
不要只说“代码报错了”，要按照以下模板给 AI 发信息：

```markdown
# 提问模板
1. 【报错信息】：复制完整的错误提示（红字部分）。
2. 【源代码】：贴出你运行的代码块。
3. 【明确要求】：请告诉我是第几行出错了？原因是什么？该如何修改？
```

### 2. AI 辅助编程技巧
* **整篇修改 vs. 局部修改**：
    * 对于简单的脚本，让 AI 把整段代码重新写一遍效率更高。
    * 对于复杂项目，要求 AI 注释出需要你手动更改的地方（例如文件路径）。
* **路径陷阱**：医学生最常犯的错是文件路径写错（如 `C:\Users\Documents\...`）。
    > **技巧**：让 AI 在代码中使用 `# TODO` 标注出你需要替换为自己电脑路径的地方。
* **长项目分段**：不要一次性把几百行代码塞给 AI。分块、分步骤喂给它，准确度会提高很多。

## 🎨 进阶篇：科研绘图与生图幻觉治理

### 1. 跨模型协作绘图
如果你没有艺术背景，可以采用 **“文字模型生成提示词 + 绘图模型生成图片”** 的工作流：

1.  **逻辑推演**：先向 GPT/Claude 描述你的医学实验场景。
2.  **生成 Prompt**：要求它将其转化为详细的英文绘图提示词。
3.  **最终成图**：将提示词复制到 **Midjourney** 或 **Stable Diffusion** 中生成。
4.  **专业组合**：可以关注 **Gemini + PicDoc** 等组合（需自行探索），实现更自动化的配图。

### 2. 缓解“生图幻觉”问题
AI 经常会画出奇奇怪怪的器官（比如六根手指或错误的解剖结构）。

* **精准描述**：提示词必须包含具体细节，如：`Style: Scientific Illustration`, `View: Cross-section`, `Anatomy tags: accurate heart valves`。
* **多次迭代**：第一张图往往不完美，需通过对话不断增加**约束条件（Constraint）**。
    * *错误示范*：“画一个心脏。”
    * *正确示范*：“请以医学插画风格画一个人类心脏左心室横截面，标注出二尖瓣，背景保持纯白干净。”
